页面置换算法

页面置换算法

当物理内存不足，需要将某个页换出到磁盘时，使用页面置换算法决定置换哪个页。

常见算法

OPT（最优置换）

置换未来最长时间不会被访问的页。理论最优，但无法实现（无法预知未来）。

FIFO（先进先出）

置换最早进入内存的页。简单但性能差，可能出现 Belady 异常。

LRU（最近最少使用）

置换最近最长时间未被访问的页。

访问序列：4 7 0 7 1 0 1 2 1 2 6
物理帧数：3

LRU 命中/缺页记录：
4: 缺页 [4]
7: 缺页 [4,7]
0: 缺页 [4,7,0]
7: 命中 [4,0,7]
1: 缺页 [0,7,1] (置换4)
...

时钟算法（Clock）

LRU 的近似实现，使用环形队列和访问位，性能较好，实际 OS 常用。

算法对比

算法缺页率实现复杂度实际应用
OPT最低无法实现理论基准
FIFO较高简单少用
LRU低复杂常用
Clock近似 LRU中等Linux 使用
