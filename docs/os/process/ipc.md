# 进程通信（IPC）

进程间通信（IPC, Inter-Process Communication）是指不同进程之间交换数据的机制。

## 常见 IPC 方式

### 管道（Pipe）

```bash
# 匿名管道：只能用于父子进程
ls | grep .md

# 命名管道（FIFO）：可用于无亲缘关系的进程
mkfifo /tmp/myfifo
```

**特点**：
- 半双工（单向传输）
- 数据只能读一次

### 消息队列

消息队列是消息的链接表，存放在内核中。

**特点**：
- 有消息边界
- 消息可以按类型接收

### 共享内存

多个进程共享同一块物理内存，速度最快。

```c
// 创建共享内存
int shmid = shmget(key, size, IPC_CREAT | 0666);
// 映射到当前进程地址空间
void *ptr = shmat(shmid, NULL, 0);
```

**特点**：
- 速度最快
- 需要同步机制（信号量）配合使用

### 信号量

用于进程/线程同步，防止竞态条件。

### Socket

可以用于不同主机间的进程通信。

## 选择建议

| 场景 | 推荐方式 |
|------|---------|
| 父子进程少量数据 | 匿名管道 |
| 无亲缘进程通信 | 命名管道/消息队列 |
| 大量数据高效通信 | 共享内存 |
| 跨主机通信 | Socket |
