# 进程调度

## 调度算法

常见的进程调度算法：

### 先来先服务（FCFS）

按照进程到达的先后顺序进行调度，非抢占式。

**优点**：简单公平
**缺点**：对短进程不友好，可能导致护航效应

### 短作业优先（SJF）

每次选择执行时间最短的进程先执行。

**优点**：平均等待时间最短
**缺点**：长进程可能饥饿

### 时间片轮转（RR）

每个进程分配一个时间片，时间片用完后切换到下一个进程。

```
进程队列：P1 → P2 → P3 → P1 → P2 → P3 ...
时间片：  [20ms][20ms][20ms][20ms]...
```

### 优先级调度

每个进程分配一个优先级，优先级高的进程先执行。

## 总结

| 算法 | 类型 | 优点 | 缺点 |
|------|------|------|------|
| FCFS | 非抢占 | 简单 | 对短进程不友好 |
| SJF | 非抢占 | 平均等待时间短 | 长进程饥饿 |
| RR | 抢占 | 公平 | 时间片设置需权衡 |
| 优先级 | 抢占/非抢占 | 灵活 | 低优先级可能饥饿 |
