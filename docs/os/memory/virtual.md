# 虚拟内存

## 为什么需要虚拟内存？

物理内存是有限的，而程序需要的内存可能超过物理内存容量。虚拟内存通过将磁盘作为内存的扩展，让程序"以为"自己拥有大量连续内存。

## 虚拟内存的实现

### 分页机制

将虚拟地址空间和物理地址空间都划分为固定大小的**页**（通常 4KB）。

```
虚拟地址 = 页号 + 页内偏移
         ↓ 页表转换
物理地址 = 帧号 + 页内偏移
```

### 页表

页表记录虚拟页号到物理帧号的映射关系。

### TLB（快表）

TLB 是页表的高速缓存，存储最近使用的页表项，加速地址转换。

## 缺页中断

当访问的页不在物理内存中时，触发**缺页中断**：

1. CPU 产生缺页中断
2. 操作系统找到缺失的页在磁盘上的位置
3. 将该页从磁盘加载到物理内存（可能需要置换其他页）
4. 更新页表，重新执行指令

## 总结

虚拟内存的核心优势：
- 允许程序使用超过物理内存大小的地址空间
- 内存保护：进程隔离
- 内存共享：多进程共享代码段
