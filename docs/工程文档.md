# LinuxArsenal 技术博客工程文档

## 一、项目概述

这是一个基于 **VuePress 2** 的静态技术博客网站，使用 `vuepress-theme-hope` 主题。主要分享嵌入式 Linux、操作系统、网络编程等技术内容。

**技术栈**：
- VuePress 2.0 (静态网站生成器)
- Vite (构建工具)
- vuepress-theme-hope (主题)
- Sass (样式预处理)
- Markdown (内容编写)

## 三、核心文件详解

### 3.1 package.json

**作用**：定义项目依赖和脚本命令

```json
{
  "name": "vuepress-starter",      // 项目名称
  "version": "1.0.0",               // 版本号
  "description": "my first vuepress project",
  "scripts": {
    "docs:dev": "vuepress dev docs",   // 开发服务器
    "docs:build": "vuepress build docs" // 生产构建
  },
  "devDependencies": {
    "@vuepress/bundler-vite": "^2.0.0-rc.26",  // Vite 构建器
    "vuepress": "2.0.0-rc.26",                   // 核心框架
    "vuepress-theme-hope": "2.0.0-rc.103",       // Hope 主题
    "sass-embedded": "^1.97.3"                   // Sass 编译器
  }
}
```

### 3.2 config.js (VuePress 配置)

**路径**：`docs/.vuepress/config.js`

**作用**：配置网站的所有行为，包括导航、侧边栏、主题等

| 配置项 | 说明 |
|--------|------|
| `base` | 站点基础路径，`/` 表示根目录 |
| `lang` | 语言设置，`zh-CN` 表示简体中文 |
| `title` | 网站标题 |
| `description` | 网站描述 (SEO) |
| `bundler` | 构建工具配置 |
| `theme` | 主题配置 |

**主题配置详解**：

```javascript
theme: hopeTheme({
    logo: '/pics/logo1.png',           // Logo 图片
    darkmode: 'toggle',                 // 暗色模式切换方式
    
    navbar: [                           // 顶部导航栏
        { text: '首页', link: '/', icon: 'home' },
        { text: '操作系统', link: '/os/', icon: 'computer' },
        { text: '嵌入式Linux', link: '/embedded/', icon: 'code' },
        { text: '关于', link: '/about/', icon: 'info' },
    ],
    
    sidebar: {...},                    // 左侧边栏
    
    pageInfo: ['Author', 'Date', 'ReadingTime', 'Category', 'Tag'],
    // 文章页面显示的信息: 作者、日期、阅读时间、分类、标签
    
    footer: 'LinuxArsenal | 嵌入式Linux技术分享',  // 页脚
    displayFooter: true,                // 显示页脚
    
    plugins: {                          // 插件配置
        markdownImage: {...},           // 图片优化插件
        markdownTab: {...},             // 标签页插件
    },
    
    markdown: {                         // Markdown 扩展
        align: true,    // 对齐
        attrs: true,    // 属性
        mark: true,     // 高亮标记
        spoiler: true,  // 折叠内容
        sub: true,      // 下标
        sup: true,      // 上标
        tasklist: true, // 任务列表
    }
})
```

**侧边栏配置结构**：

```javascript
sidebar: {
    '/os/': [                           // 匹配 /os/ 路径
        {
            text: '进程管理',           // 分组标题
            icon: 'process',            // 图标
            prefix: '/os/process/',      // 子路径前缀
            collapsible: true,           // 是否可折叠
            children: [
                { text: '进程与线程', link: 'intro' },
                // link: 'intro' 会解析为 /os/process/intro
            ],
        },
    ],
}
```

### 3.3 docs.yml (GitHub Actions 部署配置)

**路径**：`.github/workflows/docs.yml`

**作用**：Push 到 main 分支时自动构建并部署到 GitHub Pages

```yaml
name: docs                              # 工作流名称

on:
  push:
    branches: [main]                   # 触发条件: main 分支推送
  workflow_dispatch:                   # 允许手动触发

jobs:
  docs:
    runs-on: ubuntu-latest             # 运行在 Ubuntu 环境
    permissions:
      contents: write                  # 写入权限 (用于部署)
    
    steps:
      - uses: actions/checkout@v4     # 拉取代码
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22             # 使用 Node.js 22
      
      - name: Install pnpm
        run: npm install -g pnpm        # 安装 pnpm 包管理器
      
      - name: Install dependencies
        run: pnpm install              # 安装项目依赖
      
      - name: Build VuePress site
        run: pnpm docs:build           # 构建静态网站
      
      - name: Deploy to GitHub Pages
        uses: crazy-max/ghaction-github-pages@v4
        with:
          target_branch: gh-pages       # 部署到 gh-pages 分支
          build_dir: docs/.vuepress/dist  # 静态文件目录
```

## 四、部署方式

### 4.1 自动部署 (推荐)

Push 代码到 `main` 分支后，GitHub Actions 自动完成：
1. 安装 Node.js 22
2. 安装 pnpm 依赖
3. 执行 `pnpm docs:build`
4. 部署 `docs/.vuepress/dist/` 到 `gh-pages` 分支

**前提条件**：
- GitHub 仓库设置中启用 GitHub Pages
- Source 选择 `gh-pages` 分支

### 4.2 本地部署

```bash
# 1. 克隆仓库
git clone https://github.com/LinuxArsenal/linuxarsenal.github.io.git
cd linuxarsenal.github.io

# 2. 安装依赖
pnpm install

# 3. 开发预览
pnpm docs:dev

# 4. 生产构建
pnpm docs:build
```

构建产物在 `docs/.vuepress/dist/`，可手动部署到任意静态服务器。

## 五、内容编写规范

### 5.1 Markdown 文件 Frontmatter

每个 `.md` 文件开头使用 YAML frontmatter：

```yaml
---
title: 页面标题              # 浏览器标签和页面标题
icon: home                   # 页面图标 (可选)
order: 1                     # 排序号 (可选)
category:                    # 分类
  - 操作系统
tag:                        # 标签
  - 进程
  - 线程
---
```

### 5.2 页面类型

| 文件位置 | 类型 | 说明 |
|---------|------|------|
| `docs/README.md` | 首页 | 使用 `home: true`，配置 Hero 和 Features |
| `docs/os/README.md` | 章节页 | 使用 `index: false`，作为目录索引 |
| `docs/os/process/intro.md` | 内容页 | 具体文章内容 |

### 5.3 首页配置 (docs/README.md)

```yaml
---
home: true                   # 启用首页模式
icon: home
title: 首页
heroText: Linux兵工厂       # 大标题
tagline: 嵌入式Linux技术分享  # 副标题
heroImage: /pics/logo1.png   # Logo 图片
actions:                     # 行动按钮
  - text: 开始学习
    link: /os/
    type: primary
features:                    # 功能卡片
  - icon: computer
    title: 操作系统
    details: 深入理解...
---

首页正文内容 (可选)
```

## 七、注意事项

1. **静态资源**：图片放入 `docs/pics/`，引用路径 `/pics/xxx.png`
2. **中文路径**：URL 建议使用英文 (如 `/os/process/intro`)
3. **清理缓存**：构建异常时删除 `.vuepress/.cache` 和 `.vuepress/.temp`
4. **主题图标**：使用 Hope 主题内置图标，参考其文档
